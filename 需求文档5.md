三、功能一：用户中心（Profile System）


---

3.1 功能说明

每个用户拥有一个“个人主页”，支持：

上传头像

修改用户名

编辑简介（bio）

查看该用户发布的博客

查看该用户的评论



---

3.2 数据库设计（扩展 user 表）

ALTER TABLE user
ADD COLUMN nickname VARCHAR(50),
ADD COLUMN avatar VARCHAR(255),
ADD COLUMN bio VARCHAR(255);


---

3.3 用户信息接口


---

获取用户信息

GET /api/user/{id}

返回：

{
  "id": 1,
  "nickname": "小明",
  "avatar": "/upload/xxx.png",
  "bio": "喜欢写代码"
}


---

修改用户信息

POST /api/user/update

{
  "nickname": "新名字",
  "avatar": "/upload/xxx.png",
  "bio": "新的简介"
}

四、功能二：评论回复（楼中楼）


---

4.1 功能说明

支持结构：

 ├── 回复1

4.2 数据库设计（关键）


---

修改 comment 表

ALTER TABLE comment
ADD COLUMN parent_id BIGINT DEFAULT NULL,
ADD COLUMN root_id BIGINT DEFAULT NULL;


---

字段解释：

字段	作用

parent_id	父评论
root_id	根评论



---

4.3 数据结构示例

id	parent_id	root_id

五、评论接口升级


---

5.1 发布评论 / 回复

POST /api/comment/add

{
  "blogId": 1,
  "content": "回复内容",
  "parentId": 2,
  "rootId": 1
}


---

5.2 获取评论（树结构）

GET /api/comment/list?blogId=1


---

后端处理逻辑

👉 查出所有评论
👉 转换为树结构（重点）


---

Java示例（核心）

Map<Long, List<Comment>> map = comments.stream()
    .collect(Collectors.groupingBy(Comment::getParentId));

for (Comment c : comments) {
    c.setChildren(map.get(c.getId()));
}

六、前端评论展示（递归组件）


---

6.1 结构

<CommentItem :comment="item" />


---

6.2 递归组件

<template>
  <div class="comment-item">
    <p>{{ comment.content }}</p >

    <button @click="reply">回复</button>

    <div class="children" v-if="comment.children">
      <CommentItem 
        v-for="c in comment.children"
        :key="c.id"
        :comment="c"
      />
    </div>
  </div>
</template>


---

6.3 回复逻辑

function reply() {
  currentParentId.value = comment.id
  currentRootId.value = comment.rootId || comment.id
}
